# Lenses — Decision Frameworks

When facing a hard decision, run it through multiple lenses. Each one surfaces
different blindspots. Disagreements between lenses are where the real insight lives.

---

## How to Set Up Your Lenses

On first boot, ask the operator:

> "Who are the people — thinkers, practitioners, mentors, historical figures —
> whose judgment you trust most? People you think of when making hard decisions?"

For each person they name, extract a lens:

1. **Core belief** — their one-line philosophy
2. **Decision heuristics** — 5-7 rules they'd apply
3. **When to use** — what situations this worldview cuts through best
4. **The question** — the one thing this person would ask your operator

Save source material (quotes, essays, interviews) to `KNOWLEDGE/sources/` so you
can reference it accurately.

---

## Lens Format

```markdown
## [Name]

**Core belief:** [One sentence]

**Decision heuristics:**
- [Rule 1]
- [Rule 2]
- ...

**When to use this lens:**
- [Situations where this worldview is most useful]

**The question they'd ask:**
*[One pointed question this person would put to the operator]*
```

---

## Using Lenses

Run the current decision through 2-3 lenses. They'll often disagree — that
tension is the point.

The goal isn't to pick one model and follow it. It's to notice which blindspot
each one exposes. A decision that holds up across conflicting lenses is usually
right. A decision that only looks good from one angle deserves more scrutiny.

---

## Populating This File

This file should be specific to the operator's actual influences, not generic
advice. An agent that gives advice framed in terms the operator genuinely
respects is more useful than one giving generic frameworks.

Once populated, reference this file whenever the operator faces a significant
decision and you want to give advice that lands rather than generic guidance.
